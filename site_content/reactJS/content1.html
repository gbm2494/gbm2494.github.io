Definition of MVC

Model

  • Contains core functionality and data.
  • It has exports procedures that perform application-specific processing.
  • It also provides functions to access its data.
  • The model is independent of specific output representations or input behavior.

View

  • Displays information to the user.
  • This information is obtained from the model.
  • Can be multiple views of the model.
  • Each view has an associated controller.
  • If one view modifies the model this change is shown in every view that uses the same model information (Publisher-Suscriber pattern).
  • Offers functionality that allows controllers to manipulate the display.

Controller

  • Handle user input received from the views as events like mouse movement, mouse buttons or keyboard input.
  • It converts events received into service requests for the model or the view.
  • Controller calls functions defined inside the model to retrieve data.
  • Is created by its specific view during view initialization (1-1 relationship).

Change-propagation mechanism

  • It allows to show most updated data.
  • Maintains a registry of the dependent components within the model.
  • Views and controllers register their need to be informed about changes.
  • Triggered by changes inside the model.
  • Only link between model and views and controllers.

MVC benefits

  • Multiple views of the same model: As MVC separates model from interfaces you can divide interfaces according to your needs, no matter you need to use the same model for multiple interfaces.
  • Synchronized views: Change-propagation mechanism allows the views to be notified about data changes and re-render its content.
  • Pluggable views and controllers: Conceptual separation of MVC allows you to exchange views and controllers that use the same model, also at runtime.
  • Independency between model and user interface: You can connect new platforms without affecting the functional core of the application. You only need suitable implementations of views and controllers for each platform.
  • Framework potential: You can create application frameworks based on this architecture pattern.

MVC cons

  • Increased complexity.
  • Potential for excessive number of updates.
  • Intimate connection between and controller.
  • Close coupling of views and controllers to a model.
  • Inefficiency of data access in view.
  • Inevitability of change to view and controlling when porting.

ReactJS Layers

React was created to fill a specific need for a specific set of technological challenges faced by Facebook. Facebook did not create a full Model-View-Controller architecture to supplant existing frameworks in order to resolve their issues creating complex user interfaces, they created React to deal with displaying data in a user interface, more specific large-scale interfaces with data that changes over time.

"This MVC structure is actually not a fair assessment of what React is or intends to be. That is because React is one particular piece of what these frameworks present. React is in its simplest form, just the view of these MVC, MVVM, or MV* frameworks."
(Gackenheimer, 2015)
