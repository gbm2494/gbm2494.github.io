What is JSX?

"JSX is a statically-typed, object-oriented programming language designed to run on modern web browsers. Being developed at DeNA as a research project, the language has following characteristics."
(DeNA, 2012)

JSX characteristics

Faster:
"JSX performs optimization while compiling the source code to JavaScript. The generated code runs faster than an equivalent code written directly in JavaScript. The gain may vary, but even the optimized JavaScript libraries like Box2D becomes faster when ported to JSX (12% faster on iOS 5.1, 29% faster on Android 2.3)."
(DeNA, 2012)
Safer:
"In contrast to JavaScript, JSX is statically-typed and mostly type-safe. The quality of applications becomes higher when being developed using JSX, since many errors will be caught during the compilation process. It also offers debugging features at the compiler level as well."
(DeNA, 2012)
Easier:
"JSX offers a solid class system much like Java, freeing the developers from working with the too-primitive prototype-based inheritance system provided by JavaScript. Expressions and statements, however, are mostly equal to JavaScript, so it is easy for JavaScript programmers to start using JSX."
(DeNA, 2012)

Pure JavaScript vs JSX Language

JavaScript pros

  • Clear language design with specification.
  • Works on any browser.
  • Very fast.
  • Small and primitive language specification.

JavaScript problems

  • Low productivity
    • Specially in medium to large-scale environments.
    • Require skills to write fast and maintainable code.
    • Cannot find bugs before execution.
    • Hard to share best practices because great variety in coding style.
    • Hard to write high-quality code.
  • Slow execution speed
    • Compared with native.
  • High memory-consuming
  • Slow startup

Reasons for JavaScript problems

  • Error checking only at runtime because it is a dynamic language.
    • This aspect impacts on productivity and quality.
  • Adaptive compilation
    • Negative impact on:
      • Execution speed
      • Startup speed
      • Memory footprint
  • No heavy compiler optimizations
    • JS JIT engines have optimizations but with limits/overheads.
    • This reason also affects execution speed.

How JSX resolves JavaScript problems?

  • A strictly-typed OO programming language
  • Convertible to JavaScript
    • Runs faster than JavaScript on web browsers.
    • Generated code is faster by optimizing it using type-info.
    • JSX is designed so that there would be no overhead when compiled to JavaScript.
  • Strict types lead to higher productivity / better quality than JavaScript
  • Interoperable with JavaScript
  • Generates source-map for debugging: technology that allows to debug server side code on web browsers written in languages other than Javascript.
  • Help writing high-quality code than in JavaScript.

JSX Basics

Primitive types (non-nullable)

  • Boolean
  • Number
  • String
  • Int (variables of this type are only generated when type is specified explicitly, reason: slow speed). Examples:
    • var n = 1; //defines a variable n of type number
    var i : int = 1; //defines a variable of type int

Nullable primitive types

  • Nullable.<boolean>
  • Nullable.<number>
  • Nullable.<string>
  • Nullable.<int>

Variant type

  • Same value as in Javascript (var).
  • Cast to another type if necessary.

Build-in Object Types

  • Object
  • Array.<T>
    • new Array.<number>; //empty array
    new Array.<number>(length); //empty array of length elements
    [ ] : Array.<boolean>; //empty array
    [1, 2, 3]; //array with 3 elements
  • Map.<T>
    • Can be created using new operator or map initialiser.
      • New Map.<number>; equals to { } : Map.<number>; //always string key
  • Boolean, Number, String as objects but is preferable to use primitive types for performance and debugging reasons.

Literals

  • Keyword literals: null (there is no undefined value as in JS), false and true.
  • Number:
    • var a = 10;
  • String:
    • var a = "foo\n";
  • RegExp:
    • var a = /[a-zA-Z0-9]+/;
  • Function (param and return types are required, unless the type can be deducted):

    • Return type required
    var g = function (n : number) : number { return n + 1;
    };
    Return type not required
    var sum = 0; [ 1, 2, 3 ].forEach(function (e) {
    sum += e;
    });
    log sum; // 6
  • Array:

    • Explicit type required
    // empty array literals require an explicit type
    var a = [] : string[];
    Explicit type not required
    // the type is clear; you can omit type declaration
    var b : string[] = [];
  • Map:

    • Explicit type required
    // empty map literals require an explicit type var a = {} : Map.<string>;
    Explicit type not required
    // the type is clear; you can omit type declaration var b : Map.<string> = {};

Operators

Same as JavaScript except following changes:

  • Type of the operands accepted by operators are more restrictive.
  • Logical operators return boolean.
  • Binary ?.
  • Introduction of as operator.
  • Delete as statement instead of operator.

Complete reference in JSX Operator Reference Oficial Site

Type Conversion

Same as JavaScript except following changes:

  • As operator: conversion between primitive types (including nullable and variant).
  • If the source type is a nullable type and if the value is null, a runtime exception is raised under debug builds. In release builds the behavior is unspecified.
  • Conversion of variant type depends on the result of typeof operator applied.

Complete reference in JSX Conversion Reference Oficial Site

Classes and Interfaces

  • Class attributes: abstract and final:

    • class Human {
       var _name : string;
       function constructor(name : string) {
        this._name = name;
       }
       function say() : void {
        log "My name is " + this._name;
       }
    }
  • Member functions:

    • Attributes for member functions (DeNA, 2012).

  • Member variables:

    • Attributes for member variables (DeNA, 2012).

    Examples:

    // ordinary variable, type and expression are optional
    attribute* var varname [: type][ = expression];

    // constant type is optional, expression is mandatory
    static const CONSTNAME [: type] = expression;

  • Member variables and functions can be called with "." notation for variables of class type or class name (abstract classes), or with this.
  • Interfaces are similar to Java, but they have abstract variables. All the non-static functions are implicitly marked as abstract, and they cannot have bodies. An interface may implement other interfaces.

Imported Files

  • You can refer to classes defined in other source files with import statement.
  • You have to import all files before first class, interface or mixin inside the file.

Examples:

// import all classes that do not start with "_" from "a.jsx"
import "a.jsx";

// import two classes C1 and _Internal from "a.jsx"
import C1, _Internal from "a.jsx";

// import all classes that do not start with "_" under prefix A
import "a.jsx" into A; // class C1 of a.jsx can be referred to as: A.C1

// import all files under directory "sub"
import "sub/*.jsx";
