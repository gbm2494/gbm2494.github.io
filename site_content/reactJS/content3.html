What is ReactJS?

"JavaScript library for building user interfaces."
(Facebook Open Source, 2017)
"React is a javascript library that produces html that is shown to user in the browser. We write components or views (snippets of code that produce html). We write multiple different components to create applications relative simple."
(Grider, 2017)
"React was built to deal with displaying data in a user interface. The difference is that React was created to serve large-scale user interfaces—Facebook and Instagram scale interfaces—with data that changes over time."
(Gackenheimer, 2015)

ReactJS Characteristics

Declarative:
"Build encapsulated components that manage their own state, then compose them to make complex UIs. Since component logic is written in JavaScript instead of templates, you can easily pass rich data through your app and keep state out of the DOM."
(Facebook Open Source, 2017)
Component-Based:
"React makes it painless to create interactive UIs. Design simple views for each state in your application, and React will efficiently update and render just the right components when your data changes. Declarative views make your code more predictable and easier to debug."
(Facebook Open Source, 2017)
Learn Once, Write Anywhere:
"We don't make assumptions about the rest of your technology stack, so you can develop new features in React without rewriting existing code. React can also render on the server using Node and power mobile apps using React Native."
(Facebook Open Source, 2017)

ReactJS Purpose

Resolve issues with user interfaces that are:

  • Fragile
  • Highly interconnected
  • Not easily maintainable

Implementation of ReactJS with pure Javascript vs JSX language

Do I have to use JSX?

  • JSX is not requirement for using React, is just a syntactic handler for calling React.createElement(component, props, ...children).
  • Anything done with JSX can be done with pure JavaScript.
With JSX
class Hello extends React.Component {
   render() {
    return <div>Hello {this.props.toWhat}</div>;
   }
}

ReactDOM.render(
   <Hello toWhat="World" />,
   document.getElementById('root')
);
Without JSX
class Hello extends React.Component {
   render() {
    return React.createElement('div', null, `Hello ${this.props.toWhat}`);
   }
}

ReactDOM.render(
   React.createElement(Hello, {toWhat: 'World'}, null),
   document.getElementById('root')
);

Why JSX instead of JS?

  • JSX has a familiar look and feel for developers and designers.
  • JSX is HTML-like, in its structure. It allows to structure your React code as you would structure your HTML.
  • JSX brings a less verbose way of doing the same JS, so it converts everything to maintainable React JavaScript.
  • JSX defines a concise and familiar syntax for defining tree structures and attributes.
  • JSX is more readable and accessible to anyone familiar with XML syntax and hierarchy.

Bundle.js file

What is bundle.js file?

ReactJS implements some JavaScript code defined on ECMAScript6 (specification standardized for JS), but as a new version no browser can't run directly this kind of code. Also, JSX can't run directly in the browser, so for that reason, React needs to use a tooling or transpile tool that can transform the code to renderable code on browser. Some libraries like webpack and babel produces this executable code on a file called bundle.js.

How does ReactJS manipulate DOM?

Virtual DOM

  • Rebuilt the interface each time the data changed or the user interacted with the application.
  • Facebook set out to change the framework from that of a set of DOM mutations each time the data changed, to what it called reconciliation.
  • Facebook did this by creating a virtual DOM that they use each time they encounter an update to calculate the minimum set of changes needed to update the application’s actual DOM.

    • How virtual DOM works on React (Li, 2015).

    Class based components

    Component state

    A key concept on React is component state, which is a JavaScript plain object to record and react to user events. When you detect your component needs to record an element in order to work you need a class based component. Every class based component has its own object state and every time the object state changes the component re-renders itself.

    Class based component

    • This type of component allows us to maintain internal record keeping.
    • Also, it allows us to handle events:
      • By declaring function which handles the event.
      • Pass the event handler to the element we want to monitor.
      • Render the method.

    Class based component syntax

    class nameComponent extends Component {
       constructor(props){
        super(props);
        this.state={nameStateItem = initialValue};
       }

       render(){
        return(
          JSX code
        );
       }

       //you can add other custom methods
       method1(){
       }
    }

    Class based component example

    import React, { Component } from 'react';

    export default class Counter extends Component {
       constructor(props){
        super(props);
        this.state={value : 0};
       }

       render(){
        return(
        <div>
        Now the counter value is: {this.state.value}
        <button onClick={this.incrementValue.bind(this)} className="btn btn-primary"> Increment</button>
        </div>
        );
       }

       incrementValue(){
        this.setState({ value: this.state.value + 1 });
       }
    }

    Class based component shown on the browser before and after some clicks.

    What is constructor function for?

    • As in other languages, constructor allows us to set state values for the component.
    • Props parameter is an object with all the properties that the component receives from other components.
    • The component can access these properties with this.props.nameProperty and this object is available in every component method.

    What is render function for?

    • This function takes a ReactElement and render it to the DOM. React knows where to place the element with a DOM element which is provided to it.
    • Render() loads at the beginning of class component loading process, no matter if it doesn't have complete data (we need to assure stable behavior when the component hasn't received data yet).
